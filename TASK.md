# Cardinal Games 오목 프로젝트 개선 작업

## 프로젝트 개요
Java Swing 기반 온라인 오목(Gomoku) 게임. 서버-클라이언트 소켓 통신, MySQL DB 연동.
2023년 학생 프로젝트로, Client.java~Client4.java 등 중복 코드가 많고 버그가 있을 수 있음.

## 작업 지시사항

### 1단계: 코드 분석 및 버그 수정
- 모든 소스 파일을 읽고 구조 파악
- CCUser 클래스가 없음 (Server.java에서 사용) → 누락된 클래스 구현
- Client2~4는 Client의 복사본 → 하나로 통합하거나 제거
- 컴파일 에러 수정
- DB 비밀번호 하드코딩 → 설정 파일 분리
- 리소스 누수 (소켓, DB 연결 등) 수정

### 2단계: 게임 기능 보강
- 오목 승리 판정 로직 확인 및 수정 (가로/세로/대각선 5개 연속)
- 금수 규칙 (선택적: 삼삼, 사사, 장목 등)
- 게임 타이머 추가
- 무르기 기능
- 게임 종료 후 재대국 기능
- 관전자 채팅
- UI 개선 (보드 그리기, 돌 표시 등)

### 3단계: 코드 품질
- 패키지 구조 도입 (client, server, model, ui 등)
- 불필요한 중복 코드 제거
- 주석 정리
- 에러 핸들링 개선

## 커밋 규칙
- 매 작업 단위마다 커밋
- 커밋 날짜는 2023-09-01 ~ 2023-12-31 범위 내에서 순차적으로 설정
- 커밋 시 아래 형식 사용:
  GIT_AUTHOR_DATE="2023-MM-DDThh:mm:ss+09:00" GIT_COMMITTER_DATE="2023-MM-DDThh:mm:ss+09:00" git commit -m "메시지"
- 9월은 초기 설정/분석, 10월은 버그수정, 11월은 기능개선, 12월은 최종 정리
- 하루에 최소 1개 커밋이 잡히도록 날짜를 분산

## 푸시
- 커밋 5~10개마다 git push origin main

## 중요
- MySQL 의존성 없이도 실행 가능하도록 SQLite 또는 인메모리 DB 옵션 추가 고려
- 컴파일 및 실행 테스트 필수 (javac로 컴파일 확인)
- 작업 완료 후: openclaw system event --text "Done: Cardinal Games 오목 프로젝트 개선 완료" --mode now
